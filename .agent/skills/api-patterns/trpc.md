# tRPC Prensipleri

> TypeScript monorepoları için uçtan uca tip güvenliği.

## Ne Zaman Kullanılır

```
✅ Mükemmel uyum:
├── Her iki uçta TypeScript
├── Monorepo yapısı
├── Dahili araçlar
├── Hızlı geliştirme
└── Tip güvenliği kritik

❌ Kötü uyum:
├── TypeScript olmayan istemciler
├── Genel (Public) API
├── REST kurallarına ihtiyaç var
└── Çoklu dil backendleri
```

## Temel Faydalar

```
Neden tRPC:
├── Sıfır şema bakımı
├── Uçtan uca tip çıkarımı
├── Yığın genelinde IDE otomatik tamamlama
├── Anında yansıyan API değişiklikleri
└── Kod üretim adımı yok
```

## Entegrasyon Desenleri

```
Yaygın kurulumlar:
├── Next.js + tRPC (en yaygın)
├── Paylaşılan tiplerle Monorepo
├── Remix + tRPC
└── Herhangi bir TS frontend + backend
```
